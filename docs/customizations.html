Custom Enhancements

Enhanced features for better UI control and dynamic updates

Dynamic Dropdown Updates

WindUI Custom supports dynamic dropdowns that can automatically fetch data from functions, ModuleScripts, or be updated manually. This is perfect for scenarios where dropdown options change based on runtime data, similar to Rayfield's dynamic dropdowns.

Automatic Data Fetching

Key Feature: When you provide a function or ModuleScript path, the dropdown will automatically fetch fresh data every time it opens, ensuring you always see the latest data. The function is called both during creation and when the dropdown menu is opened.

Using a Function

-- Dropdown that fetches players dynamically
-- The function is called when dropdown is created AND every time it opens
local playerDropdown = Tab:Dropdown({
    Title = "Select Player",
    Desc = "Fetches players from game (updates on open)",
    Values = function()
        local players = {}
        for _, player in ipairs(game:GetService("Players"):GetPlayers()) do
            table.insert(players, {
                Title = player.Name,
                Icon = "user"
            })
        end
        -- Return placeholder if no players found
        if #players == 0 then
            return {{ Title = "--" }}
        end
        return players
    end,
    Callback = function(option)
        print("Selected player:", option.Title)
    end
})

-- Optional: Manually refresh data from function
-- playerDropdown:RefreshData()
-- Note: Function is automatically called when dropdown opens, so RefreshData() is usually not needed

Important: The function is automatically called every time the dropdown menu opens, ensuring you always see the latest data. You don't need to manually call :RefreshData() unless you want to refresh while the dropdown is closed.

Using a ModuleScript

-- ModuleScript: MyDataModule (in ReplicatedStorage, ServerStorage, etc.)
-- Can return a table or a function that returns a table
-- Option 1: Return a table directly
-- return {
-- { Title = "Item 1" },
-- { Title = "Item 2" }
-- }
--
-- Option 2: Return a function
-- return function()
-- local items = getItemsFromGame()
-- return items
-- end

local itemDropdown = Tab:Dropdown({
    Title = "Select Item",
    Desc = "Fetches items from ModuleScript (updates on open)",
    Values = "MyDataModule", -- Searches for ModuleScript in game
    Callback = function(option)
        print("Selected:", option.Title)
    end
})

-- Optional: Manually refresh data from ModuleScript
-- itemDropdown:RefreshData()
-- Note: ModuleScript is automatically accessed when dropdown opens

ModuleScript Behavior: If the ModuleScript returns a function, that function will be called every time the dropdown opens. If it returns a table, that table will be used. The dropdown automatically searches common game locations for the ModuleScript.

Manual Updates

You can also manually update dropdown values using :SetValues(). This is useful when you want to update the dropdown programmatically, regardless of the original data source:

local dropdown = Tab:Dropdown({
    Title = "Dynamic Dropdown",
    Values = {
        { Title = "Option 1", Icon = "file" },
        { Title = "Option 2", Icon = "folder" },
    },
    Callback = function(option)
        print("Selected:", option.Title)
    end
})

-- Later, update the dropdown values
-- Selection is preserved if the same option still exists
dropdown:SetValues({
    { Title = "New Option A", Icon = "star" },
    { Title = "New Option B", Icon = "heart" },
    { Title = "New Option C", Icon = "zap" },
})

-- Or refresh from the original data source (if function/ModuleScript)
-- dropdown:RefreshData()

Note: :SetValues() preserves the current selection if the selected option still exists in the new values. If the selection no longer exists, it will be cleared.

Features

  • Automatic refresh on open: Function-based Values are called every time the dropdown opens, ensuring fresh data
  • Automatic data fetching from functions or ModuleScripts
  • Preserves current selection when possible after updates
  • Works with both single-select and multi-select dropdowns
  • Handles open menus gracefully during updates
  • Clears search filters when values are refreshed
  • Automatically recalculates list and canvas sizes
  • RefreshData() method to manually refetch from original source
  • SetValues() method for manual updates with new data

Data Source Locations

When using a ModuleScript path (string), WindUI searches in these locations:

  • ReplicatedStorage
  • ServerStorage
  • StarterPlayerScripts
  • StarterCharacterScripts
  • Workspace

Window Outline Customization

You can add a customizable outline around the main window frame. The outline can be enabled/disabled and its color, thickness, and transparency can be customized both during window creation and at runtime.

During Window Creation
-- Create window with outline enabled (default)
local Window = WindUI:CreateWindow({
    Title = "My Script",
    Author = "by Developer",
    Folder = "myscript",
    Icon = "settings",
    OutlineEnabled = true, -- Enable outline (default: true)
    OutlineColor = Color3.fromHex("#30ff6a"), -- Outline color (default: white)
    OutlineThickness = 2, -- Outline thickness in pixels (default: 1)
    OutlineTransparency = 0.5 -- Outline transparency 0-1 (default: 0.8)
})

-- Create window with outline disabled
local Window = WindUI:CreateWindow({
    Title = "My Script",
    Folder = "myscript",
    OutlineEnabled = false -- Disable outline
})
At Runtime

You can change the outline properties at any time using the following methods:

-- Change outline color
Window:SetOutlineColor(Color3.fromHex("#60a5fa"))
-- Outline color updates immediately!

-- Change outline thickness
Window:SetOutlineThickness(3) -- Set thickness to 3 pixels
-- Outline thickness updates immediately!

-- Change outline transparency
Window:SetOutlineTransparency(0.3) -- Set transparency to 0.3 (0 = opaque, 1 = fully transparent)
-- Outline transparency updates immediately!

-- Enable or disable outline
Window:SetOutlineEnabled(true) -- Enable outline
Window:SetOutlineEnabled(false) -- Disable outline

-- All methods return the Window object for method chaining
Window:SetOutlineColor(Color3.fromHex("#ff6a30"))
    :SetOutlineThickness(2)
    :SetOutlineTransparency(0.5)
    :SetOutlineEnabled(true)
Configuration Parameters
Parameter Type Default Description
OutlineEnabled boolean true Whether the outline is enabled. Set to false to disable the outline.
OutlineColor Color3 Color3.fromRGB(255, 255, 255) The color of the outline around the window frame.
OutlineThickness number 1 The thickness of the outline in pixels. Recommended range: 1-5 pixels.
OutlineTransparency number 0.8 The transparency of the outline. Range: 0 (fully opaque) to 1 (fully transparent).
Methods
  • Window:SetOutlineColor(color) - Set the outline color. Returns the Window object for chaining.
  • Window:SetOutlineThickness(thickness) - Set the outline thickness in pixels. Returns the Window object for chaining.
  • Window:SetOutlineTransparency(transparency) - Set the outline transparency (0-1). Returns the Window object for chaining.
  • Window:SetOutlineEnabled(enabled) - Enable or disable the outline. Returns the Window object for chaining.

Window Outline Shadow Customization

You can add a customizable shadow around the window outline to create a depth effect. The shadow is attached to the main window frame as a child, so it automatically moves and resizes with the window - no manual syncing needed! The shadow can be enabled/disabled and its color, transparency, and thickness can be customized both during window creation and at runtime.

Quick Start: How to Implement Outline Shadow

To add a shadow to your window outline, simply enable it when creating your window:

-- Step 1: Enable the outline (required for shadow)
-- Step 2: Enable the shadow
local Window = WindUI:CreateWindow({
    Title = "My Script",
    Folder = "myscript",
    OutlineEnabled = true, -- Required: Enable outline first
    OutlineShadowEnabled = true -- Enable shadow
})

-- That's it! The shadow will appear automatically.
-- You can customize it further with the options below.

Important: The outline must be enabled (OutlineEnabled = true) for the shadow to work. If the outline is disabled, the shadow will not be visible.

During Window Creation
-- Create window with outline shadow enabled
local Window = WindUI:CreateWindow({
    Title = "My Script",
    Author = "by Developer",
    Folder = "myscript",
    Icon = "settings",
    OutlineEnabled = true,
    OutlineShadowEnabled = true, -- Enable outline shadow (default: false)
    OutlineShadowColor = Color3.fromHex("#000000"), -- Shadow color (default: black)
    OutlineShadowTransparency = 0.5, -- Shadow transparency 0-1 (default: 0.5)
    OutlineShadowThickness = 3, -- Shadow thickness in pixels (default: uses OutlineThickness)
    OutlineShadowOffset = 3 -- Shadow offset/blur in pixels (default: 2)
})
Complete Implementation Example
-- Full example with all shadow customization options
local Window = WindUI:CreateWindow({
    Title = "My Script",
    Author = "by Developer",
    Folder = "myscript",
    Icon = "settings",
    
    -- Outline settings (required for shadow)
    OutlineEnabled = true,
    OutlineColor = Color3.fromHex("#ffffff"),
    OutlineThickness = 2,
    OutlineTransparency = 0.8,
    
    -- Shadow settings
    OutlineShadowEnabled = true,
    OutlineShadowColor = Color3.fromHex("#000000"), -- Black shadow
    OutlineShadowTransparency = 0.5, -- 50% transparent
    OutlineShadowThickness = 3 -- 3 pixels thick (or nil to match outline)
})

-- The shadow is now active and will move with the window automatically!
At Runtime

You can enable or modify the shadow properties at any time using the following methods:

-- Enable outline shadow
Window:SetOutlineShadowEnabled(true)
-- Shadow appears immediately!

-- Change shadow color
Window:SetOutlineShadowColor(Color3.fromHex("#ff0000"))
-- Shadow color updates immediately!

-- Change shadow transparency
Window:SetOutlineShadowTransparency(0.3) -- Make shadow more visible
-- Shadow transparency updates immediately!

-- Change shadow thickness
Window:SetOutlineShadowThickness(4) -- Set shadow thickness to 4 pixels
-- Shadow thickness updates immediately!

-- Note: Shadow offset is reserved for future use
-- The shadow is attached to the window and moves with it automatically

-- All methods return the Window object for method chaining
Window:SetOutlineShadowEnabled(true)
    :SetOutlineShadowColor(Color3.fromHex("#000000"))
    :SetOutlineShadowTransparency(0.4)
    :SetOutlineShadowThickness(3)
Configuration Parameters
Parameter Type Default Description
OutlineShadowEnabled boolean false Whether the outline shadow is enabled. Set to true to enable the shadow.
OutlineShadowColor Color3 Color3.fromRGB(0, 0, 0) The color of the outline shadow. Typically black for a natural shadow effect.
OutlineShadowTransparency number 0.5 The transparency of the shadow. Range: 0 (fully opaque) to 1 (fully transparent). Lower values create more visible shadows.
OutlineShadowThickness number nil (uses OutlineThickness) The thickness of the shadow outline in pixels. If not specified, uses the same thickness as the main outline. Recommended range: 1-10 pixels.
OutlineShadowOffset number 2 The visual offset/blur amount of the shadow. Note: The shadow is attached to the main window frame and moves with it automatically. This parameter is reserved for future use. Recommended range: 1-10 pixels.
Methods
  • Window:SetOutlineShadowEnabled(enabled) - Enable or disable the outline shadow. Creates the shadow frame if it doesn't exist. Returns the Window object for chaining.
  • Window:SetOutlineShadowColor(color) - Set the shadow color. Returns the Window object for chaining.
  • Window:SetOutlineShadowTransparency(transparency) - Set the shadow transparency (0-1). Returns the Window object for chaining.
  • Window:SetOutlineShadowThickness(thickness) - Set the shadow thickness in pixels. If set to nil, uses the main outline thickness. Returns the Window object for chaining.
  • Window:SetOutlineShadowOffset(offset) - Reserved for future use. The shadow is attached to the window frame and moves with it automatically. Returns the Window object for chaining.
Implementation Tips
  • Enable Outline First: Always set OutlineEnabled = true before enabling the shadow. The shadow requires an outline to be visible.
  • Automatic Movement: The shadow is attached to the window frame, so it automatically moves and resizes with the window. No manual syncing needed!
  • Color Recommendations: Black shadows (Color3.fromHex("#000000")) work best for a natural depth effect, but you can use any color.
  • Transparency Range: Lower transparency values (0.2-0.4) create more visible shadows. Higher values (0.6-0.8) create subtle shadows.
  • Thickness Matching: If you don't specify OutlineShadowThickness, it will automatically match your OutlineThickness.
  • Performance: The shadow has minimal performance impact since it's a simple UIStroke attached to the window frame.
Troubleshooting
-- ❌ WRONG - Shadow won't show if outline is disabled
local Window = WindUI:CreateWindow({
    OutlineEnabled = false, -- Outline disabled
    OutlineShadowEnabled = true -- Shadow won't be visible!
})

-- ✅ CORRECT - Enable outline first
local Window = WindUI:CreateWindow({
    OutlineEnabled = true, -- Outline enabled
    OutlineShadowEnabled = true -- Shadow will work!
})

Note: The outline shadow is attached as a child of the main window frame, so it automatically moves and resizes with the window. No manual syncing is needed - the shadow stays perfectly aligned with the outline when dragging or resizing the window. The shadow is positioned behind the main window frame (ZIndex = 0) to create a depth effect.

Colorpicker Integration

Colorpickers can automatically display the current UI colors by using the ThemeProperty parameter. When you provide a theme property name, the colorpicker will automatically fetch and display the current color from the theme (including any custom overrides) before you make changes.

-- Colorpicker that displays the current Accent color
Tab:Colorpicker({
    Title = "Accent Color",
    Desc = "Customize the accent color",
    ThemeProperty = "Accent", -- Automatically gets current Accent color
    Callback = function(color)
        WindUI:SetCustomAccent(color)
        -- UI updates immediately!
    end

-- Colorpicker that displays the current Background color

    end

Refreshing Color Picker Displays

Important: After loading a config or programmatically changing colors, you may need to refresh your color picker displays to show the updated colors. The color picker's visual display (the small colored square) may not automatically update when colors are changed externally.

-- Method 1: Refresh individual color picker using Update() method
local accentColorPicker = Tab:Colorpicker({
    Title = "Accent Color",
    ThemeProperty = "Accent",
    Callback = function(color)
        WindUI:SetCustomAccent(color)
    end
})

-- After loading a config or changing color programmatically:
local Creator = Window.WindUI.Creator
if Creator and Creator.GetThemeProperty then
    local currentColor = Creator.GetThemeProperty("Accent", Creator.Theme)
    if currentColor and accentColorPicker then
        -- Extract Color3 if it's a gradient table
        local colorToSet = currentColor
        if typeof(currentColor) == "table" and currentColor.Color then
            colorToSet = currentColor.Color
        end
        -- Update the color picker display
        accentColorPicker:Update(colorToSet)
    end
end

-- Method 2: Refresh all color pickers at once (recommended after loading configs)
local function refreshAllColorPickers()
    task.wait(0.3) -- Wait for theme to update
    local allColorPickers = {
        accentColorPicker,
        textColorPicker,
        buttonColorPicker,
        -- ... add all your color pickers here
    end
    
    for _, colorPicker in ipairs(allColorPickers) do
        if colorPicker and colorPicker.ThemeProperty then
            pcall(function()
                local Creator = Window.WindUI.Creator
                if Creator and Creator.GetThemeProperty then
                    local currentColor = Creator.GetThemeProperty(
                        colorPicker.ThemeProperty,
                        Creator.Theme
                    )
                    
                    if currentColor then
                        -- Handle gradient tables
                        local colorToSet = currentColor
                        if typeof(currentColor) == "table" and currentColor.Color then
                            colorToSet = currentColor.Color
                        end
                        
                        -- Update the color picker display
                        if colorPicker.Update then
                            colorPicker:Update(colorToSet, colorPicker.Transparency)
                        elseif colorPicker.Set then
                            colorPicker:Set(colorToSet, colorPicker.Transparency)
                        end
                    end
                end
            end)
        end
    end
end

-- Call after loading a config:
ConfigManager:LoadConfigByName("myconfig")
task.spawn(function()
    refreshAllColorPickers()
end)

When to refresh:

  • After loading a config: When you load a config that contains custom colors, refresh all color pickers to show the loaded colors
  • After programmatic color changes: If you change colors using SetCustomAccent() or similar methods programmatically (not through the color picker UI), refresh the displays
  • After theme changes: If you switch themes or apply theme overrides, refresh to show the new colors

Note: Color pickers automatically refresh when you click on them (if they have ThemeProperty set), but the visual display square may not update until you refresh it manually or click on the color picker.

Immediate Updates

All SetCustom methods update the UI immediately when called. You don't need to manually refresh or rebuild any UI elements. The library automatically applies changes to all existing UI elements that use the affected theme properties.

-- Example: Dynamic color changes

        local hue = i / 100
        local color = Color3.fromHSV(hue, 1, 1)
        WindUI:SetCustomAccent(color)
        -- UI updates immediately with each color change!

    end
end)

Automatic Config Saving & Loading

All custom theme colors are automatically saved and loaded with configs! You don't need to manually register or save them - the config system handles everything automatically.

How It Works

When you use any SetCustom* method (like SetCustomAccent(), SetCustomDropdownSelected(), etc.), the color is stored in the theme system's override table. When you save a config, the system automatically:

  1. Scans all custom overrides: Collects all colors you've set using any SetCustom* method
  2. Serializes the colors: Converts Color3 and Gradient objects to JSON-safe format
  3. Saves to config file: Stores them in the __themeOverrides field of the config

When you load a config, the system automatically:

  1. Reads saved overrides: Retrieves all custom colors from the config file
  2. Deserializes the colors: Converts them back to Color3 or Gradient objects
  3. Applies to UI: Restores all colors and updates the UI immediately
-- User customizes their UI with custom colors
WindUI:SetCustomAccent(Color3.fromHex("#30ff6a"))
WindUI:SetCustomBackground(Color3.fromHex("#1a1a1a"))
WindUI:SetCustomDropdownSelected(Color3.fromHex("#60a5fa"))
WindUI:SetCustomText(Color3.fromHex("#ffffff"))
WindUI:SetCustomButton(Color3.fromHex("#52525b"))

-- User can also use gradients - they're saved too!
WindUI:SetCustomBackground(WindUI:Gradient({
    ["0"] = { Color = Color3.fromHex("#0f0c29"), Transparency = 0 },
    ["100"] = { Color = Color3.fromHex("#302b63"), Transparency = 0.1 },
}, {
    Rotation = 45,
}))

-- Save config - ALL custom colors are automatically included!
local ConfigManager = Window.ConfigManager
local configFile = ConfigManager:CreateConfig("mytheme")

-- No need to register colors - they're saved automatically!
-- Just register UI elements if you want to save their values
configFile:Register("myToggle", myToggle)
configFile:Register("mySlider", mySlider)

-- Save - custom colors are automatically saved!
configFile:Save()
-- ✅ All custom colors (Accent, Background, DropdownSelected, etc.) are saved!
-- ✅ Both Color3 and Gradient colors are properly saved!

-- Later, load the config
ConfigManager:LoadConfigByName("mytheme")
-- ✅ All custom colors are automatically restored!
-- ✅ UI updates immediately with all the saved colors!
-- ✅ Gradients are properly restored!

What Gets Saved

Every custom theme color you set is automatically saved, including:

  • All colors set with SetCustomAccent(), SetCustomBackground(), SetCustomText(), etc.
  • All colors set with SetCustomDropdownSelected() and other component-specific methods
  • All colors set with SetCustomProperty() for any custom property name
  • Both Color3 and Gradient types are fully supported

Important Notes

  • No Registration Required: Custom theme colors don't need configFile:Register() - they're automatically included when you call configFile:Save()
  • Complete Persistence: Users can have different color schemes for different configs - each config saves its own set of custom colors
  • Immediate Application: When loading a config, all colors are applied immediately - the UI updates automatically
  • Gradient Support: Complex gradients with multiple stops and rotation are fully preserved

Example: Multiple Configs with Different Themes

-- Create "Dark Theme" config
WindUI:SetCustomAccent(Color3.fromHex("#30ff6a"))
WindUI:SetCustomBackground(Color3.fromHex("#0a0a0a"))
WindUI:SetCustomText(Color3.fromHex("#ffffff"))
local darkConfig = ConfigManager:CreateConfig("darktheme")
darkConfig:Save()
-- ✅ Dark theme colors saved!

-- Create "Light Theme" config
WindUI:SetCustomAccent(Color3.fromHex("#0066ff"))
WindUI:SetCustomBackground(Color3.fromHex("#f5f5f5"))
WindUI:SetCustomText(Color3.fromHex("#000000"))
local lightConfig = ConfigManager:CreateConfig("lighttheme")
lightConfig:Save()
-- ✅ Light theme colors saved!

-- Switch between themes by loading different configs
ConfigManager:LoadConfigByName("darktheme")
-- ✅ Dark theme colors applied!

ConfigManager:LoadConfigByName("lighttheme")
-- ✅ Light theme colors applied!
