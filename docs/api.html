API Reference

WindUI:CreateWindow

Creates a new window container for your UI.

Parameters

Parameter Type Description
Title string Window title displayed in the topbar
Author string Author name displayed below title
Folder string Folder name for saving configs
Icon string Icon name for the window
Size UDim2 Optional custom window size
NewElements boolean Use new element styling (default: false)
HideSearchBar boolean Hide the search bar (default: false)
OpenButton table Configuration for the open button

Example

local Window = WindUI:CreateWindow({
    Title = "My Script",
    Author = "by Developer",
    Folder = "myscript",
    Icon = "settings",
    NewElements = true
})

Methods

Window:Tab(config)

Creates a new tab in the window.

Window:Section(config)

Creates a section directly in the window.

Window:Tag(config)

Adds a tag to the window topbar.

Window:Destroy()

Destroys the window and all its elements.

Tab:Dropdown

Creates a dropdown menu for selecting options.

Parameters

Parameter Type Description
Title string Dropdown label
Desc string Optional description
Values table/function/string Array of option tables, function returning table, or ModuleScript path. Function is called on creation AND every time dropdown opens.
DataSource function/string Custom Enhancement: Alternative to Values for dynamic data. Function or ModuleScript path that provides dropdown options.
Value string/table Initial selection (string for single, table for multi)
Multi boolean Enable multi-select (default: false)
AllowNone boolean Allow deselecting all options (default: false)
SearchBarEnabled boolean Enable search functionality (default: false)
MenuWidth number Custom menu width
Locked boolean Disable the dropdown (default: false)
Callback function Function called when selection changes: function(option)

Methods

Dropdown:SetValues(newValues)

Custom Enhancement: Updates dropdown options dynamically. If newValues is nil and a DataSource exists, it will refresh from the original source. Preserves selection when possible. Automatically refreshes UI, clears search filters, and recalculates sizes.

Parameters: newValues (table) - Array of option tables. If empty, uses placeholder {{Title = "--"}}.

Returns: boolean - true if successful

Dropdown:RefreshData()

Custom Enhancement: Refetches data from the original DataSource (function or ModuleScript). Useful for manually refreshing while dropdown is closed. Note: Function-based Values are automatically refreshed when dropdown opens, so this is usually not needed.

Returns: boolean - true if successful

Dropdown:LockValues(values)

Locks specific values from being selected.

Dynamic Data Sources

Custom Enhancement: The Values parameter can be:

  • Table: Static list of options (default behavior)
  • Function: Function that returns a table of options - called automatically on creation AND every time the dropdown opens to ensure fresh data
  • String: ModuleScript name/path - searches game and requires the module. If module returns a function, it's called on each open.

Key Feature: When using a function or ModuleScript that returns a function, the dropdown automatically fetches fresh data every time it opens, ensuring you always see the latest options. You can also manually refresh with :RefreshData().

-- Function-based Values (automatically called on open)
local dropdown = Tab:Dropdown({
    Title = "Players",
    Values = function()
        -- This function is called when dropdown opens
        local players = {}
        for _, p in ipairs(game.Players:GetPlayers()) do
            table.insert(players, {Title = p.Name})
        end
        return players
    end
})

Parameters: name (string) - Property name, value (Color3, Gradient, or number)
